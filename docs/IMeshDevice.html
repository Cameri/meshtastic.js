IMeshDevice

Parent class for all connection interfaces - do not call directly

Constructor

(abstract) new IMeshDevice()

Source:
  • imeshdevice.js, line 20
Fires:
  • event:fromRadio - Gets called whenever a fromRadio message is received from device,event: returns fromRadio data
  • event:dataPacket - Gets called when a data packet is received from device
  • event:userPacket - Gets called when a user packet is received from device
  • event:positionPacket - Gets called when a position packet is received from device
  • event:nodeListChanged - Gets called when node database has been changed,event: returns changed node number
  • event:connected - Gets called when link to device is connected
  • event:disconnected - Gets called when link to device is disconnected
  • event:configDone - Gets called when device has been configured (myInfo, radio and node data received).event: Device interface is now ready to be used
